HNSW Index — Production Speed

What is HNSW?

Hierarchical Navigable Small World graph

• Connects similar vectors in a multi-layer graph
• Search by greedy graph traversal
• Incremental: add vectors without rebuild
• Persists to disk — build once, query forever

HNSW vs HRM2: When to use what

HNSW — Default for production
0.995+ recall, 591 QPS on 100K

HRM2 Splats — When uncertainty matters
Overlap-based similarity, confidence scores

Rule of thumb: Use HNSW for speed, HRM2 when uncertainty matters

Vector Compression

TurboQuant

Random projection + quantization
3–8 bit per dimension
No training data, no codebooks
8x compression on 640D

PolarQuant

Polar coordinate quantization
3–8 bit per dimension
Best for directional similarity
>95% recall at 5% search fraction
640D vector: 2,560 bytes → 320 bytes (4-bit TurboQuant)

Gaussian Splatting for Search

From 3D neural rendering to vector search — each document becomes a probability distribution.

μ, α, κ — Beyond Points

Every document stores three values — not just coordinates, but a probability distribution.

μ — Mean

Position in latent space. Like any embedding, but it's the center of a distribution.

α — Opacity

How much this document "matters". High α = landmark. Low α = uncertain / noisy.

κ — Concentration

How focused. High κ = one precise topic. Low κ = broad, covers many concepts.

ELI5

"Golden Retriever puppy" = high κ. "Dogs in general" = low κ. Agents know when to trust results.

TurboQuant & PolarQuant

Compress vectors 3-8× with minimal recall loss.

TurboQuant

Random projection + binary coding. 4-bit ≈ 3% recall loss for 8× less memory.

PolarQuant

Polar coordinate quantization. Better for 768D+. Preserves angular distance.

Direct Search

Search compressed codes — no decompression. quant-index → quant-search.

Hot-Swap

Change bits/algorithms without re-ingesting. Re-quantize on the fly.

Spatial Memory: MemPalace

Organize memory like physical space — Wings, Rooms, Halls, Tunnels.

Wing = Project / Persona
├── Room = Theme (auth, billing, migration)
│ ├── Hall = Type (facts, decisions, events)
│ └── Hall = Type (errors, warnings)
└── Tunnel ── connects shared rooms across wings

Query: "auth decisions from project-x"
1. Filter wing → 10% corpus → 2. Filter room → 2% → 3. Vector search → high recall

Verbatim Storage

"Store everything verbatim, never let an LLM decide what to remember"

Splat

μ, α, κ — probabilistic representation. Fast search, compressed.

Closet (AAAK)

30× compressed text any LLM reads natively. ~40 tokens vs ~1200.

Drawer (Verbatim)

Original text. Never summarized. Source of truth.

Flow

Agent reads Closet → decides relevance → fetches Drawer. Saves ~90% tokens.

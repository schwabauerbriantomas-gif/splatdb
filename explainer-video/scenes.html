The Problem

Vector Search Loses Confidence

Every AI app needs nearest-neighbor search. But no engine tells you how sure it is.

Flat Index

O(N) brute force — exact but unusable past 10K. 100K vectors = 94ms/query.

HNSW

Fast graph search but zero confidence info. Is the result 99% or 60% certain? You can't tell.

Product Quantization

8-bit compression destroys distance semantics. ~3% recall loss with no way to measure it.

The Missing Piece

No engine captures uncertainty — how certain are we that this match matters?

Core Concept

Gaussian Splatting × Search

From 3D neural rendering to vector search — each document becomes a probability distribution

Three Parameters

μ · α · κ

Not just a point — a probability distribution per document

μ — Mean Position

Center of the distribution in latent space. Like any embedding, but with context.

α — Opacity

How much this doc "matters". High α = landmark. Low α = noise. Auto-computed from data density.

κ — Concentration

How focused the doc is. High κ = one precise topic. Low κ = broad, covers many concepts.

Why It Matters

Agents get confidence scores. "95% sure this is relevant" vs "maybe related" — act on certainty.

Compression

TurboQuant & PolarQuant

3-8× compression with minimal recall loss

TurboQuant

Random projection + binary coding. 4-bit → 8× memory reduction, ~3% recall loss.

PolarQuant

Polar coordinate quantization. Better for 768D+ vectors. Preserves angular distance.

Direct Search

Search compressed codes without decompression. quant-index → quant-search.

Hot-Swap

Change bits/algo without re-ingesting. Re-quantize on the fly.

MemPalace

Spatial Memory

Organize memory like physical space — wings, rooms, halls

Wing = Project / Persona / Domain ├── Room = Theme (auth, billing, migration) │ ├── Hall = Type (facts, decisions, events) │ └── Hall = Type (errors, warnings) ├── Room = Theme (payments) │ └── Hall = Type (facts, events) └── Tunnel ── connects shared rooms across wings Query: "auth decisions from project-x" 1. Filter wing → ~10% of corpus 2. Filter room → ~2% of corpus 3. Filter hall → ~0.5% of corpus 4. Vector search → high recall, minimal noise

Storage

Verbatim Storage

"Store everything verbatim, never let an LLM decide what to remember"

Splat (Gaussian)

μ, α, κ — probabilistic representation. Fast search, compressed.

Closet (AAAK Summary)

30× compressed text any LLM reads natively. ~40 tokens instead of ~1200.

Drawer (Verbatim)

Original document text. Never summarized. Source of truth.

Flow

Agent reads Closet → decides if worth it → fetches Drawer. Saves ~90% tokens.
